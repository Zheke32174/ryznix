Goal (set 2026-06-15): a full functional Linux in `/data/local/tmp` with **no root, no proot, no bootloader unlock**. Device SM-S948U, Android 16/SDK36, locked bootloader — true uid 0 is impossible (carrier policy), so the ceiling is `shell` (uid 2000) via Shizuku `~/rish`. This achieves that ceiling.

**How it runs without root:** binaries are Termux/bionic ELFs whose interp is `/system/bin/linker64` (always present on Android). Launched as `shell` via rish, they execute from `/data/local/tmp` (the one dir `shell` owns + can exec). NOT glibc/Ubuntu — `os-release` says tmp-linux/Termux. User explicitly chose this over a glibc rootfs.

**THE key blocker was never root — it was SELinux, not DAC.** app uid (`untrusted_app`, what plain Termux Bash runs as) can READ Termux's `usr/lib` but CANNOT write `/data/local/tmp` even at mode 0777 (SELinux denies untrusted_app→shell_data_file). `shell` (rish) can write tmp but can't read Termux's per-app-category files. Neither identity can do both. 777 "dropboxes" do nothing. **Bridge = /sdcard (FUSE):** both domains may touch it. app uid `tar`s the source → `/sdcard/x.tar` → rish extracts with `/data/local/tmp/busybox tar -xf` (busybox avoids depending on the very libs being staged; tar preserves symlink chains FUSE can't).

**Working layout** (`/data/local/tmp`): `bin/` = symlink farm (1607) into `termux/bin` + `system/bin`; `lib/` = full 6.5G Termux usr/lib copy (staged via bridge); `system/usr/` = prior tree holding **python3.15 stdlib + headers + pip**; `termux/` = prior bionic mirror; `include/` = staged usr/include (191M) + `asm→asm-generic` shim + minimal `asm-generic/types.h` for compiles. Config in `/data/local/tmp/env.sh` (PATH, LD_LIBRARY_PATH spanning lib+termux/lib+system/usr/lib, PYTHONHOME=/data/local/tmp/system/usr, SSL_CERT_FILE=system/usr/etc/tls/cert.pem). Launcher: `/data/local/tmp/enter.sh` → sources env, execs bash. Enter with: `echo '. /data/local/tmp/env.sh; exec bash' | ~/rish` or `~/rish` then `/data/local/tmp/enter.sh`.

**Verified working (2026-06-15):** bash 5.2, python3.15+pip25.3 (ssl/sqlite/hashlib), node v24, go1.26.3, php8.5, git (real commits), curl 8.17 (HTTPS rc=200, TLS verified, DNS via /data/local/tmp/etc/resolv.conf), openssl, cmake, OpenSSH10.3, rsync, jq, vim, tmux, htop, sqlite3, xz, zstd. **C compiles + runs**: `gcc --sysroot=/data/local/tmp -I/data/local/tmp/include hello.c` → runnable bionic ELF. ~half the alphabetical bin/ entries are broken wrapper scripts (hardcoded bad paths) — cosmetic, the compiled tools work. **SUPER LINUX layer (Ubuntu glibc, added 2026-06-15):** on top of the bionic userland, real **Ubuntu 24.04.4 LTS glibc** runs in `/data/local/tmp/ubuntu` with **no root, no proot, no userns** (unprivileged userns is BLOCKED on this Samsung kernel — `unshare` returns EINVAL, no `unprivileged_userns_clone` knob; that's *why* ELF-conversion is the only path). Launcher: `/data/local/tmp/superlinux.sh` (PATH = ubuntu/usr/bin + tmp/bin both worlds; LD_LIBRARY_PATH = bionic libs only).
